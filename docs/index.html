Summary

We plan to simulate fluids in a 3D space. By changing the density of the fluid, we hope to see different physical dispersions of the liquid. Alternatively, should this prove more difficult than expected, we will use an open source fluid simulator and focus on changing the fluid properties, such as density and viscosity.

Problem Description

Fluid simulation is a special case of physical simulation. By extending the 2D lattice structure from project 4 to 3D, we can simulate interactions of particles in 3 dimensions.

This is challenging because we are representing a fluid as an infinite number of particles. We would need to find the minimum number of particles needed for a realistic representation. The constraints for each particle are non-linear and include components of the n neighbors of each particle. This makes it hard to solve the equations via linear methods and may cause instability in our solution if momentum is not conserved.

3D fluid simulation is cool! Animation of fluids is common in video games and animated movies. We think it’s important for simulating natural disasters (tsunamis & floods), and fluid simulation appears in other places as water/fluids are omnipresent.

We plan to implement this by modeling a fluid as a lot of particles. Then, we apply forces to each particle at every stage in a loop. As these particles move, they exert forces on each other, causing the movement of the entire fluid. We adjust the density of these particles to affect how quickly the fluid moves.

Goals and Deliverables

What we Plan to Deliver

Our end goal will be to successfully implement the Nvidia research paper on water simulation by Macklin and Muller, with the particles colored using a height map gradient. As a backup plan, should this prove to be more difficult than we expect, we will pull an open source fluid simulation and focus on adding bells and whistles, such as path tracing, density controls, and other things. We expect to get an A given either a successful implementation of the Nvidia paper or a successful implementation of bells and whistles added to an open source fluid simulation.

Should we succeed in implementing the Nvidia research paper on water simulation by Macklin and Muller, we plan to add a density control that would allow us to demonstrate different types of fluids, ranging from water vapor (gases) to lava (the most dense liquid we could think of). We will aim to create GIFs of simulated fluids of different densities to demonstrate the differences in simulation due to the fluid density. We will also aim to color the particles in the simulation using a height-based gradient. We believe we can achieve this because it will be an extension of project 4 to 3 dimensions, which means that we have the necessary tools and resources to get started easily. We also have learned enough about physical simulation in lecture to implement the paper's equations effectively. However, if this proves time consuming, we have the knowledge from class to implement path tracing and density controls on an existing fluid simulation.

What we Hope to Deliver (Stretch Goals)

  • Interactive Slider bar to change the density of the fluid while it is being simulated in realtime.
  • Use path tracing to render the fluid surface, perhaps using ellipsoid splatting (as done in the Nvidia paper), or using techniques from project 3.

Metrics for Quality

We will measure the quality of the simulation by comparing it to the baselines in videos of moving fluids and to output images in the Nvidia research paper. We may also run our fluids through a similar simulator implemented in C that we found in open source to compare the outcomes.

Analysis Questions

In our analysis, we plan to answer the following questions:

  • How can the particle model of physical simulation be applied to simulating 3D surfaces ie. fluids?
  • How can we efficiently search for neighboring particles to display realistic particle motion in real-time?
  • How can we efficiently render a fluid surface using path tracing/ellipsoid splatting? (stretch)
